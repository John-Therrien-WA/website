{%- comment -%} Fixed, full-bleed wallpaper for a wallpaper-host page. See the "Wallpaper pages" block in styles/main.css and in _includes/animation.html for the mechanism. Pair this with a wallpaper-host whose data-wallpaper selector is ".{name}-wallpaper" — both derive from the same name, so they can't drift. Params: name — image basename in assets/images/. e.g. "musings" pulls musings-wallpaper-{400,800,1414}.webp + musings-wallpaper-1414.jpg (the standard variant set produced by scripts/build-images.js). width — intrinsic width of the fallback img (default 1414). height — intrinsic height of the fallback img (default 2000). A page gets its own background by pointing its `wallpaper` front matter at a different basename and adding that image set — no markup change here. {%- endcomment -%} {%- assign wp = include.name -%} {%- assign base = '/assets/images/' | append: wp | append: '-wallpaper-' -%}
